Added / clarified
Note that the full run uses 25 jobs by default (not 10).
--debug: use when fetch finds only 1 URL or all jobs are “blocked” (e.g. --phase fetch --max-jobs 10 --debug).
--phase pdf: regenerate only PDFs from existing .md in output/resumes/ (e.g. after changing the template).
--headless: run browser in background (commented, since it can increase blocks).
--top-n: limit how many top jobs appear in the report (e.g. --top-n 15).
